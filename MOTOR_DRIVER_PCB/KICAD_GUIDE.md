# KiCad 학습 가이드 — MDD10A×2 + Mega 신호 분배 PCB

> 진행 방법: 각 Phase의 **[왜]**를 먼저 읽고 → 직접 KiCad에서 작업 → 저장 후
> Claude에게 "Phase N 끝났어, 검토해줘"라고 하면 `.kicad_sch`/`.kicad_pcb` 파일을
> 텍스트로 읽고 실수를 짚어준다. (KiCad 파일은 전부 텍스트라 코드 리뷰처럼 검토 가능)

---

## 이 보드의 존재 이유 (설계 근거)

브레드보드의 문제 3가지를 없애는 것이 목적이다:

1. **접촉 불량** — 브레드보드 스프링 접점은 로봇 주행 진동에 빠진다. PCB는 납땜/커넥터 고정.
2. **분배점 부족** — GND가 필요한 곳: 드라이버 2개(신호GND) + 엔코더 4개 + Mega = 7곳.
   5V는 엔코더 4개. 브레드보드 레일로는 지저분해지는 걸 PCB 구리 트레이스가 대신한다.
3. **배선 실수** — 실크스크린에 핀 이름을 인쇄해두면 케이블을 꽂을 때 헷갈리지 않는다.

**이 보드에 모터 전원(24V)은 올리지 않는다.** 이유: 모터 전류(최대 10A/채널)는
배터리 → 드라이버 B+/B− 터미널로 굵은 전선 직결이 안전하고, 이 보드는 수십 mA의
로직 신호만 다루므로 얇은 트레이스로 충분해진다. (검증 완료: 배터리+ → B+,
배터리− → B−, B− → Mega GND 공통접지 — 현재 배선 그대로 유지)

## 확정 넷리스트 (펌웨어 = 진실의 원천)

`urt/mecanum_motor/src/main.cpp`의 핀맵과 1:1 대응. **PCB가 코드를 따라간다.**

| 신호 | Mega 핀 | 목적지 | 왜 이 핀인가 |
|------|---------|--------|--------------|
| PWM_M1 | 5 | 드라이버1 PWM1 | 펌웨어가 Timer3/4로 5kHz PWM 생성하는 핀 |
| PWM_M2 | 6 | 드라이버1 PWM2 | 〃 |
| PWM_M3 | 7 | 드라이버2 PWM1 | 〃 |
| PWM_M4 | 8 | 드라이버2 PWM2 | 〃 |
| DIR_M1~M4 | 22,23,24,25 | 각 드라이버 DIR1/DIR2 | 방향은 아무 디지털핀이나 가능. Mega 끝 2열 헤더에 몰아 배선 정리 |
| ENC_M1~M4 | 2,3,18,19 | 엔코더 B채널 (단독 사용) | **외부 인터럽트 전용 핀** — 펌웨어 attachInterrupt가 여기서만 펄스 카운트. 엔코더의 사용 채널(B)은 반드시 이 핀으로 |
| +5V | 5V | 엔코더 4개 전원 | 홀센서 엔코더 급전(개당 ~10-20mA, USB 예산 내) |
| GND | GND | 드라이버 2개 + 엔코더 4개 | 공통 기준전위. 없으면 PWM HIGH/LOW 판정 불가 |

### 커넥터 구성 (총 10개)

| 이름 | 핀수 | 내용 (핀 순서) |
|------|------|----------------|
| J1 MEGA_PWM | 1×4 | PWM_M1, PWM_M2, PWM_M3, PWM_M4 |
| J2 MEGA_DIR | 1×4 | DIR_M1..M4 |
| J3 MEGA_ENC | 1×4 | ENC_M1..M4 (→ Mega 2,3,18,19 인터럽트 핀) |
| J4 MEGA_PWR | 1×2 | +5V, GND |
| J5 DRV1 | 1×5 | **DIR1, PWM1, DIR2, PWM2, GND** ← 실물 헤더와 같은 순서 |
| J6 DRV2 | 1×5 | 〃 |
| J7~J10 ENC1~4 | 각 1×3 | +5V, GND, ENC_Mx (엔코더 B채널) |

> 설계 변경 이력: 엔코더는 B채널만 사용하기로 결정(2026-07) → 커넥터 1×4→1×3,
> ENCA/ENCB 구분 폐지, 신호명 ENC_Mx로 통일. 트레이드오프: 진짜 쿼드러처(외력·
> 슬립 시 방향 실측)는 불가 — 필요해지면 보드 리비전. 펌웨어의 방향 추정 방식
> (명령 부호 기반)과는 일치함.

J5/J6 순서를 실물과 똑같이 하는 이유: **케이블이 1:1 스트레이트**가 되어
꼬임/실수가 없다. (드라이버 실물 실크로 검증 완료: DIR1/PWM1/DIR2/PWM2/GND)

넷 연결 규칙:
- PWM_M1: J1.1 ↔ J5.PWM1 / PWM_M2: J1.2 ↔ J5.PWM2 / PWM_M3,M4는 J6으로
- DIR도 같은 패턴. ENC_Mx: J3.x ↔ J(6+x).3
- +5V: J4.1 ↔ 모든 ENC.1 / GND: J4.2 ↔ J5.5, J6.5, 모든 ENC.2

## Phase 1 — 회로도: 심볼 배치

**[왜 회로도부터?]** 회로도 = "무엇이 무엇과 연결되는가"(논리),
PCB = "어디에 어떤 폭의 구리로"(물리). 분리하는 이유: 논리 정확성을 먼저
기계 검사(ERC)로 확정하고, 물리 배치는 나중에 자유롭게 바꿔도 논리가 깨지지
않게 하기 위해서다. 소프트웨어의 "인터페이스 정의 → 구현" 분리와 같다.

- [ ] 회로도 편집기 열기 (`.kicad_sch` 더블클릭)
- [ ] `A` 키 → `Conn_01x04` 검색 (Connector_Generic 라이브러리) → 배치
- [ ] 같은 방법으로: `Conn_01x08` ×1 (J2), `Conn_01x04` ×5 (J1,J3,J7~J10),
      `Conn_01x02` ×1 (J4), `Conn_01x05` ×2 (J5,J6)
- [ ] 각 심볼 더블클릭 → Reference(J1..J10)와 Value(MEGA_PWM 등) 입력

단축키: `M` 이동, `R` 회전, `Esc` 취소. 심볼은 왼쪽에 Mega측(J1~J4),
오른쪽에 출력측(J5~J10)을 놓으면 신호 흐름(왼→오)이 읽기 좋다.

**[왜 Conn_01x0N 심볼?]** 이 보드의 "부품"은 커넥터뿐이다. Arduino Mega 자체는
이 보드에 실장되지 않으므로 심볼이 필요 없다 — 케이블 건너편에 있을 뿐.

## Phase 2 — 회로도: 라벨로 배선

**[왜 선 대신 라벨?]** 커넥터 10개 사이에 실선을 다 그으면 거미줄이 된다.
같은 이름의 **넷 라벨(net label)**은 자동으로 같은 넷 — 선을 안 그어도 연결된다.
가독성이 좋고, 이름 자체가 문서가 된다.

- [ ] `L` 키 → 라벨 이름 입력(위 표의 신호명 그대로: `PWM_M1`, `DIR_M1`, `ENCA_M1`...)
      → 해당 핀 끝에 부착 (핀에서 짧은 선을 `W` 키로 뽑고 그 끝에 라벨)
- [ ] 전원은 전원 심볼 사용: `A` → `+5V`, `GND` 검색 → J4와 각 엔코더/드라이버 GND 핀에 배치

**[왜 전원 심볼?]** `+5V`/`GND` 심볼은 전역 연결 + 도면에서 전원 흐름이 한눈에
보이는 관례다. GND 삼각형이 아래로, +5V 화살표가 위로 — 회로도 읽는 문법.

## Phase 3 — ERC (전기 규칙 검사)

**[왜?]** 발주 후 발견하는 실수 1건 = 2주 + 배송비. ERC는 미연결 핀,
중복 라벨 오타(`PWM_M1` vs `PWN_M1`)를 공짜로 잡는다.

- [ ] Inspect → Electrical Rules Checker → Run
- [ ] **"Input Power pin not driven by any Output Power pins" 에러가 나면 정상!**
      → `A` → `PWR_FLAG` 심볼을 +5V와 GND 넷에 하나씩 붙인다.

**[왜 PWR_FLAG?]** KiCad는 "이 전원 넷에 실제로 전력을 공급하는 부품이 있는가"를
검사한다. 이 보드의 5V는 **보드 밖(Mega)에서** 들어오므로 공급원이 도면에 없다.
PWR_FLAG는 "외부에서 급전됨을 내가 보증한다"는 선언이다. 커넥터-온리 보드의
필수 통과의례이자 초보자가 가장 많이 검색하는 에러.

## Phase 4 — 풋프린트 지정

**[왜 심볼과 풋프린트가 분리?]** 논리(4핀 커넥터)와 물리(수직 핀헤더? JST?
터미널블록?)는 별개 결정이다. 분리 덕에 같은 회로도로 물리 구현만 바꿀 수 있다.

- [ ] Tools → Assign Footprints
- [ ] 전 커넥터: `Connector_PinHeader_2.54mm` 라이브러리에서
      `PinHeader_1x0N_P2.54mm_Vertical` 선택 (N = 핀수)

**[왜 2.54mm 핀헤더?]** 듀폰(점퍼) 케이블과 그대로 호환 — 지금 가진 케이블 재사용.
트레이드오프: 진동에 빠질 수 있다. 로봇이 험하게 달리면 나중에 락 달린
JST-XH(같은 2.5mm급)로 바꾸는 선택지가 있다 — 회로도는 그대로, 풋프린트만 교체.

## Phase 5 — PCB로 전송 + 배치

**[왜 배치가 라우팅보다 중요?]** 배치가 좋으면 라우팅은 저절로 풀리고,
배치가 나쁘면 어떤 라우팅도 스파게티가 된다.

- [ ] 회로도에서 `F8` (Tools → Update PCB from Schematic) → PCB 편집기로 부품 전송
      (이것이 옛날의 "네트리스트 내보내기/가져오기"를 대체한 현대 KiCad 방식)
- [ ] 배치 원칙: **커넥터는 전부 보드 가장자리**(케이블이 꽂히는 곳),
      왼쪽 변에 J1~J4(Mega측), 오른쪽 변에 J5/J6(드라이버), 아래 변에 J7~J10(엔코더)
- [ ] Edge.Cuts 레이어 선택 → 사각형 보드 외곽선 그리기 (대략 80×60mm면 여유)
- [ ] **M3 마운팅 홀 4개** 추가: `A`는 PCB에선 안 되니 풋프린트 직접 배치 —
      Add Footprint(`A`) → `MountingHole_3.2mm_M3` 4모서리

**[왜 마운팅 홀?]** 로봇은 진동한다. 나사 고정 없는 PCB는 케이블 힘으로 흔들려
납땜 크랙이 난다. 브레드보드를 벗어나는 이유 1번이 진동이었음을 기억.

## Phase 6 — 라우팅 + 넷 클래스

**[왜 폭이 다른가?]** 트레이스는 전선이다. 굵기 = 허용 전류.
경험칙(1oz 구리): **1A당 약 0.25mm**. 신호(수 mA)는 0.3mm도 과분하고,
5V/GND(엔코더 4개 합산 ~100mA)도 0.5mm면 5배 이상 여유. 넓게 쓰는 이유는
전류가 아니라 **제조 수율과 납땜 내구성**.

- [ ] File → Board Setup → Net Classes:
      `Default` 0.3mm / 새 클래스 `Power` 0.5mm 만들고 `+5V`, `GND` 넷 할당
- [ ] `X` 키로 라우팅 시작 — 흰색 가는 선(ratsnest)이 "아직 안 이은 연결"
- [ ] 신호선부터 전부 라우팅. GND는 **일부러 다 잇지 않아도 된다** (다음 Phase가 해결)

## Phase 7 — GND 카퍼 푸어

**[왜?]** 뒷면 전체를 GND 구리판으로 채우면: ① GND 배선이 저절로 완성되고
② 모터 PWM 노이즈(로봇의 최대 적)가 엔코더 신호로 새는 크로스토크를 줄이며
③ 리턴 전류가 항상 최단 경로를 찾는다.

- [ ] 뒷면(B.Cu) 선택 → Add Filled Zone (`Ctrl+Shift+Z`) → 넷 = GND → 보드 전체 둘러 그리기
- [ ] `B` 키 = 존 채우기(refill). 라우팅 수정 후엔 항상 `B` 다시.
- [ ] 패드 연결이 X자(thermal relief)로 보이는 건 **정상** — solid로 바꾸지 말 것.
      (통구리에 직결하면 납땜 열이 다 빨려서 냉납이 된다)

## Phase 8 — 실크스크린

**[왜?]** 3개월 뒤의 나는 남이다. 커넥터 옆에 이름이 인쇄돼 있으면
케이블 꽂을 때 회로도를 다시 열 필요가 없다.

- [ ] F.Silkscreen 레이어 → 텍스트(`Ctrl+Shift+T`)로 각 커넥터 옆에:
      `MEGA PWM 5,6,7,8` / `DRV1 → MDD10A#1` / `ENC1 5V,G,A,B` 식으로
- [ ] 1번 핀 위치 표시(점 또는 삼각형) — 케이블 뒤집어 꽂기 방지

## Phase 9 — DRC → 거버 → 발주

**[왜 DRC?]** 제조사 장비는 최소 트레이스 간격/폭 공차가 있다. DRC는
"이 보드가 물리적으로 제조 가능한가"의 기계 검사. ERC가 논리 검사였다면 DRC는 물리 검사.

- [ ] Inspect → Design Rules Checker → 에러 0 만들기
- [ ] File → Fabrication Outputs → Gerbers:
      레이어 F.Cu, B.Cu, F/B.Silkscreen, F/B.Mask, Edge.Cuts 체크,
      **Use Protel filename extensions** 체크 → Plot
- [ ] 같은 창에서 **Generate Drill Files** 버튼 (Excellon, mm) — **잊으면 구멍 없는 보드가 온다.** 최다 빈발 실수.
- [ ] 출력 폴더를 zip → JLCPCB/PCBWay 업로드 → 2층, 1.6mm, 수량 5 (최소수량, ~$2+배송)
